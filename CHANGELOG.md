# 更新日志

本文档记录个人简历网站的重要调整，便于后续维护和面试复盘。

## 2026-06-21

### Added

- 新增透明人物图 `avatar-professional-cutout-soft.webp`，用于首页 Hero 区。
- 首页 Hero 右侧改为透明人物图 + CSS 背景组合，减少整图拼贴感。
- 项目详情页支持 Markdown 内容渲染。
- 新增统一的 Markdown 内容样式，包括标题、段落、列表和行高。
- 新增项目卡片结构规范，包括图片区、标题区、描述区、标签区和底部操作区。
- README 文档补充网站定位、页面结构、技术栈、核心项目和部署说明。
- 新增 `PROJECTS.md`，用于项目复盘和面试准备。
- 新增 `DEPLOYMENT.md`，用于本地预览和部署维护。

### Changed

- 统一全站 CSS 版本号，减少浏览器缓存导致的旧样式问题。
- 优化浅色主题下的导航、按钮、卡片、标签、代码面板和 Hero 区显示。
- 优化首页人物边缘、背光、投影和底部暗部过渡。
- 优化移动端 Hero 区代码卡片位置，避免右侧裁切。
- 将主导航改为 `nav > ul > li > a` 结构，提升 HTML 语义化。
- 将章节标题容器从普通 `div.section-head` 调整为 `header.section-head`。
- 重构 `js/main.js`，集中选择器、文案和存储 key，拆分主题、导航、Markdown、卡片和联系表单逻辑。
- 统一卡片、标签、正文、链接等 CSS 变量，降低深色 / 浅色主题维护成本。
- 清理 `output/` 验证产物和未使用的中间图片资源。

### Fixed

- 修复破图和旧版人物素材残留问题。
- 修复浅色背景下按钮、代码卡片和标签对比度不足的问题。
- 修复手机端代码卡片在浅色模式下被右侧裁切的问题。
- 修复时间线日期横线与标题距离过近的问题。
- 修复项目卡片标题、正文、标签和底部操作区布局不稳定的问题。
- 修复 Markdown 内容直接作为普通文本渲染导致的标题、段落和列表间距异常。

### Removed

- 删除 `output/` 临时截图和浏览器缓存文件。
- 删除未被页面引用的中间图片：
  - `assets/avatar-professional-cutout.webp`
  - `assets/avatar-professional-transparent.webp`
