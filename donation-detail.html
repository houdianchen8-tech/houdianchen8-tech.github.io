项目背景

校园中存在大量闲置物品（教材、电子产品等），这些物品往往在毕业季被丢弃。传统跳蚤市场受限于信息渠道，流转效率低。因此设计并实现了该系统，用于提升物品流转管理效率和用户体验。

系统架构

系统整体采用前后端分离的分层架构（Controller → Service → DAO）设计。

基本流程：用户通过浏览器访问前端页面 → 前端调用后端 RESTful API → 后端进行参数校验、业务处理和权限判断 → 数据通过 MySQL 进行持久化存储 → 系统返回处理结果并在前端展示。

技术难点与解决方案

难点：物品列表多表关联查询响应慢

问题描述：在开发过程中，物品列表页需要关联用户表、分类表、订单状态表进行多表查询，随着数据量增长，页面加载时间超过 3 秒。

解决方案：

  • 开启 MySQL 慢查询日志，捕获瓶颈查询
  • 使用 EXPLAIN 分析执行计划，定位到多表关联查询未走索引
  • 为高频查询字段（物品状态、分类 ID、发布时间）添加联合索引
  • 引入分页查询机制，控制单次返回数据量

最终效果：优化后列表查询时间由 3.2 秒降至 0.4 秒，查询效率提升约 30%。
