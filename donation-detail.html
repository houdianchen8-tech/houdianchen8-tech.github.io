高校物品捐赠系统

围绕校园闲置物品流转效率低的问题，基于 Spring Boot + MySQL 设计并实现了一套完整的物品捐赠管理系统。负责系统整体架构设计、数据库建模及核心业务逻辑实现，通过 SQL 性能优化将查询效率提升约 30%。

项目背景

校园中存在大量闲置物品，如教材、电子产品、生活用品等，这些物品往往在毕业季被丢弃或闲置。与此同时，很多同学又有获取这些物品的需求。传统的校园跳蚤市场受限于信息传播渠道，流转效率低下。

该项目旨在通过信息化手段解决这一痛点，设计一套完整的物品捐赠管理系统，实现物品发布、申请、审核、配送和评价的全流程管理。

多级分类结构，支持按类别、按热度、按时间等多维度排序。

性能优化实践

在开发过程中遇到物品列表查询响应慢的问题，通过以下步骤定位并解决：

  • 开启 MySQL 慢查询日志，捕获瓶颈查询
  • 使用 EXPLAIN 分析执行计划，定位到多表关联查询未走索引
  • 添加联合索引覆盖查询字段，优化表关联顺序
  • 引入分页查询机制，控制单次返回数据量
  • 优化后查询效率提升约 30%

设计亮点

  • 采用 RESTful 风格设计接口，语义化 URL 与标准 HTTP 方法
  • 基于 Token 实现无状态权限控制，支持移动端与 Web 端共用
  • 模拟百级并发场景验证系统稳定性和响应性能
  • 分层架构设计，模块间解耦，便于后续功能扩展
