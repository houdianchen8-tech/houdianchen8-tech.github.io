## 1. 高校物品捐赠系统

### 项目定位

面向校园师生的闲置物品捐赠管理系统，解决校园闲置物品流转效率低、信息分散、交易闭环弱的问题。

### 我的角色

- 项目负责人
- 后端开发
- 数据库设计
- 核心业务逻辑实现
- 性能优化

### 技术栈

- Java
- Spring Boot
- Spring Security
- MyBatis-Plus
- MySQL
- Redis
- JWT
- WebSocket
- RESTful API

### 核心功能

- 用户注册与管理员审核
- JWT 鉴权与多角色权限隔离
- 物品发布、审核、下架、匹配、捐出、完成
- 捐赠申请与交易交付
- 双方互评信用机制
- WebSocket 实时通知
- 管理端统计与反馈处理

### 技术难点

#### 多表关联查询响应慢

物品列表页需要关联用户、物品、分类、申请记录等多张表。随着数据量增长，列表查询耗时超过 3 秒。

解决方式：

- 开启 MySQL 慢查询日志定位瓶颈 SQL。
- 使用 `EXPLAIN` 分析执行计划。
- 为高频字段添加联合索引。
- 引入分页查询，限制单次返回数据量。
- 对热点推荐结果使用 Redis 缓存。

结果：

- 核心列表查询由 `3.2s` 优化至 `0.4s`。
- 响应时间降低约 `87.5%`。

### 面试可讲点

- 为什么选择前后端分离架构？
- JWT 鉴权如何设计？
- 用户、管理员、志愿者 3 种角色如何隔离权限？
- 物品状态和交易状态为什么分开设计？
- 联合索引如何选择字段顺序？
- Redis 缓存如何设置 key、过期时间和更新策略？
- WebSocket 通知失败时如何兜底？

## 2. 智慧校园 AI 助手

### 项目定位

基于 RAG 架构的校园 AI 问答系统，面向课程信息、校园公告、办事流程等多源数据，帮助师生快速获取信息。

### 我的角色

- 后端核心开发者
- 数据服务模块开发
- 知识检索链路构建
- Prompt 与上下文拼接优化
- 接口性能优化

### 技术栈

- Java
- Spring Boot
- Redis
- MySQL
- RAG
- LLM
- Prompt Engineering
- Cursor

### 核心流程

1. 接收用户自然语言问题。
2. 进行问题解析与意图识别。
3. 从知识库召回相关文档片段。
4. 对召回结果进行排序、去重和上下文拼接。
5. 构造 Prompt 调用大模型。
6. 返回答案并记录日志。

### 技术难点

#### 回答质量不稳定

同样的问题在不同轮次可能得到不同质量的回答，偶尔出现遗漏关键信息的问题。

解决方式：

- 优化 Prompt 模板，明确角色、边界和输出格式。
- 优先拼接相关性更高的文档片段。
- 对检索结果进行去重与排序。
- 增加缓存策略，减少重复请求。
- 建立异常处理和兜底提示。

结果：

- 接口延迟降低约 `35%`。
- 模块开发周期缩短约 `40%`。
- 回答稳定性和关键信息覆盖率提升。

### 面试可讲点

- RAG 和普通 LLM 问答有什么区别？
- 知识库如何切分和清洗？
- 召回结果如何排序？
- Prompt 模板如何设计？
- 多轮对话上下文如何保留？
- 为什么使用 Redis？
- AI 回答错误时如何兜底？
