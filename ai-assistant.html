项目概述

智慧校园 AI 助手是一个面向校园师生的 AI 智能问答系统，基于 RAG（检索增强生成）架构设计。系统实现对课程信息、校园公告、办事流程等多源校园数据的语义检索与智能应答，帮助师生快速获取所需信息。

项目采用 Java / Spring Boot 构建后端服务，Redis 实现向量检索与缓存，MySQL 存储结构化数据，通过大语言模型驱动自然语言问答。我在实习期间作为后端核心开发者，负责数据服务模块开发、知识检索链路构建与系统优化工作。

项目背景

校园信息分散在网站、公告栏和各个部门系统中，师生查找信息效率低下，主要面临以下问题：

  • 课程信息发布不集中，学生需逐个平台查找，耗费大量时间
  • 校园公告更新速度快，关键信息容易被大量通知淹没
  • 办事流程不统一，师生需多次咨询相关部门，沟通成本高

为了解决上述痛点，团队决定构建一套基于 AI 的智能问答系统，将分散的多源校园数据统一接入、结构化处理，并通过语义检索与自然语言交互降低师生获取信息的门槛。

系统架构

系统采用 RAG（检索增强生成）架构设计，核心流程从用户输入问题开始，经过意图识别、向量检索、上下文拼接，最终由大模型生成精准回答。

完整流程：用户输入问题 → 意图识别与预处理 → 从 Redis 向量库检索相关文档片段 → 检索结果与用户问题拼接为 Prompt → 调用大模型生成回答 → 返回结果并记录对话上下文

技术难点与解决方案

难点：AI 回答不稳定 / 质量波动

同样的提问在不同轮次得到的回答质量不一致，有时会遗漏关键信息，影响用户体验。

解决方案：

  • 优化 Prompt 模板结构：明确角色设定、输出格式和约束条件，减少自由生成的不确定性
  • 改进上下文拼接策略：优先召回相关性最高的前 K 条文档片段，确保关键信息优先进入 Prompt
  • 引入检索结果去重与排序机制，减少噪声信息对模型生成的干扰
  • 实施缓存策略 + 异步请求优化，接口响应延迟降低约 35%

最终效果：回答质量与稳定性显著提升，关键信息覆盖率达到预期目标。

优化成果

通过系统性优化，项目在性能与开发效率方面取得显著成果：

  • 缓存策略 + 异步请求优化 → 接口响应延迟降低约 35%
  • Prompt 模板优化 → 回答质量与稳定性明显提升
  • Cursor 辅助编码 → 模块开发周期缩短约 40%
  • 完备的监控兜底机制 → 多用户并发场景稳定运行

项目最终实现了 4 大核心模块（数据服务、向量检索、上下文工程、多轮对话），接入多源校园数据，开发 15+ 个后端接口。

项目总结

通过该项目，我深入实践了 RAG 架构的完整实现流程——从多源数据接入与向量化、检索策略设计，到 Prompt 工程与模型调用封装，积累了 AI 应用从 0 到 1 的落地经验。项目不仅提升了我对检索增强生成技术的理解，也让我掌握了在真实业务场景中进行性能优化和质量保障的方法。
